# Lesson Outline

- Lesson title: Neural Networks in Practice
- Lesson goal: turn neural experiments into reproducible engineering projects
- Anchor case: support-ticket classification
- Main skill or harness: ML Neural Project
- Primary visual: project structure around the model
- On-screen artifact: config, split rules, logging plan, checkpoint plan
- Viewer takeaway: a neural project becomes real when another engineer can reproduce and debug it

## Teaching Flow

1. Open with the danger of notebook-only training.
2. Show the minimum structure of a reproducible neural project.
3. Explain logging, validation, and checkpointing.
4. Show the debug checklist that should exist before scaling up.
5. End with the Neural Project harness.
