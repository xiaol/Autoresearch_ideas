# Lesson Outline

- Lesson title: Unsupervised Learning, Embeddings, and Retrieval
- Lesson goal: show how retrieval quality becomes inspectable engineering evidence
- Anchor case: company knowledge search
- Main skill or harness: ML Retrieval Evaluation
- Primary visual: query -> retrieved results -> relevance judgment
- On-screen artifact: retrieval review sheet plus runnable example output
- Viewer takeaway: retrieval systems should be judged by representative queries and failure inspection, not demo vibes

## Teaching Flow

1. Open with a search task that seems to work until hard queries appear.
2. Explain embeddings and retrieval at a systems level.
3. Run the tiny retrieval example.
4. Show relevance definition, cutoffs, failures, and diagnosis.
5. End with the Retrieval Evaluation harness.
