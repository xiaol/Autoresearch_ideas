# Lesson Outline

- Lesson title: Evaluation, Error Analysis, and Experiment Design
- Lesson goal: make evaluation the center of engineering judgment
- Anchor case: fraud metrics with threshold and slice tradeoffs
- Main skill or harness: ML Evaluation Review
- Primary visual: metric -> hidden tradeoff -> action consequence
- On-screen artifact: evaluation review memo
- Viewer takeaway: a model result is only useful when you know what it hides

## Teaching Flow

1. Open with a model result that sounds strong but may mislead.
2. Show why metric-task fit matters.
3. Explain imbalance, thresholding, calibration, and slices.
4. Connect experiment design to trustworthy conclusions.
5. End with the Evaluation Review harness.
