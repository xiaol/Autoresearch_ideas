# Scene Cards

## Scene 1

- Purpose: show silent data breakage
- On screen: field name stays the same, meaning changes
- Narration beat: the model can degrade while the schema still looks normal

## Scene 2

- Purpose: define key data risks
- On screen: schema, timestamp, join, freshness, lineage
- Narration beat: types are not enough; meaning and timing matter

## Scene 3

- Purpose: explain train-serve mismatch
- On screen: offline feature path versus live path
- Narration beat: the model may train on a world it never sees at runtime

## Scene 4

- Purpose: show the readiness review
- On screen: review checklist or readiness memo
- Narration beat: serious modeling begins after trust is earned

## Scene 5

- Purpose: close with the harness
- On screen: readiness verdict
- Narration beat: narrow honest trust is better than broad false trust
