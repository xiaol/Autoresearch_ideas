# Lesson Outline

- Lesson title: Data, Labels, and Problem Framing
- Lesson goal: show that many ML failures begin before the first model
- Anchor case: support-ticket escalation and fraud labels
- Main skill or harness: ML Problem Framer
- Primary visual: decision -> prediction moment -> label -> feature availability
- On-screen artifact: product brief or task memo
- Viewer takeaway: project names are not task definitions

## Teaching Flow

1. Open with a vague ML request that sounds reasonable but is underspecified.
2. Separate business target from prediction target.
3. Define the prediction moment and label-generation process.
4. Show leakage and proxy-label risks.
5. End with the Problem Framer skill and the framing memo artifact.
