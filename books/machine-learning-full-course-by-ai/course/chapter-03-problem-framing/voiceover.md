# Voiceover Draft

## Opening

Many machine learning projects fail before the first model trains. They fail because the team never defined the real decision clearly enough.

## Core Explanation

A project title like churn prediction or fraud detection is not a task definition. We still need to know the decision supported, the prediction moment, the label process, and which features are actually available at that moment.

## Risk Layer

Labels are not pure truth. They are produced by systems, delays, investigators, workflows, and incentives. That means a model may learn the habits of the labeling system instead of the underlying phenomenon.

## Skill Connection

The ML Problem Framer skill exists to slow the team down before they rush into model families and benchmarks.

## Close

A clear task is not bureaucracy. It is the first serious model improvement.
