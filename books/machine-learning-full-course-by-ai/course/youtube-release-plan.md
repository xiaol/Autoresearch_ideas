# YouTube Release Plan

This book is best released as both a structured series and a later full-course compilation.

## Publishing Strategy

Use a two-layer release model:

1. Publish each chapter as its own lesson video.
2. After the chapter set is stable, publish a combined full-course video.

This gives you faster feedback, more discoverability, and a clean path to the final long-form course.

## Core Release Set

- 16 core chapter videos
- 3 bonus videos from Appendix A, Appendix B, and Appendix C
- 1 final combined course upload

## Recommended Naming Pattern

Use a stable pattern so the playlist looks coherent:

`Machine Learning Full Course by AI | Chapter 01 | Harness Engineering`

Keep the chapter number in the visible title so the sequence is obvious.

## Description Structure

Each description should include:

- the chapter promise in one sentence
- the real case shown in the lesson
- the book chapter reference
- the runnable example path if one exists
- the relevant skill name
- the course playlist link

## Thumbnail and Title Card Rule

Use one visual identity across the whole course:

- same typography system
- same chapter numbering format
- part-based color coding
- one short promise line

The generated `assets/title-card.svg` files are the starting point, not the final limit.

## Recommended Rollout

If you want steady audience growth:

1. Release 2 to 3 foundational chapters first.
2. Release one or two lessons per week.
3. Group chapters into playlist sections by course part.
4. Publish the combined course after the chapter videos have already earned search traffic.

If you want a faster big launch:

1. Finish all chapter assets first.
2. Record the full sequence in batches.
3. Release the full-course version.
4. Cut the long video into chapter clips afterward.

## What To Measure

Track:

- click-through rate on title and thumbnail
- first 30-second retention
- retention at the demo section
- comments requesting follow-up examples
- which chapters send viewers to the runnable examples and skills

## Practical Recommendation

For this project, the strongest path is:

1. finish the chapter asset packs
2. record the 16 chapter lessons
3. publish them as a playlist
4. then publish the full combined course

That sequence matches how the repository is already structured.
