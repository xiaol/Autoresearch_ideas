# Glossary

## AI-native learning

Learning designed around collaboration with AI tools, while preserving human responsibility for verification, judgment, and synthesis.

## harness engineering

The design of the full working environment around a human-AI task so that reusable skills, tools, memory, evals, and review checkpoints produce more reliable outcomes.

## baseline

A simple reference model or method used to judge whether a more complex approach is actually better.

## calibration

The degree to which predicted confidence matches real-world accuracy.

## capstone

A larger project that integrates concepts from multiple chapters into one meaningful artifact.

## concept compression

The act of reducing a large topic into a small, faithful explanation without losing the essential structure.

## data leakage

Any accidental path by which information from evaluation or future data leaks into training and produces misleadingly good results.

## distribution shift

A change between the data used during development and the data seen in production.

## derivative

The local rate at which an output changes when an input changes slightly.

## embedding

A dense numerical representation that captures useful structure about an item, token, user, or document.

## error analysis

The structured study of where, how, and why a model fails.

## evaluation loop

A repeated process of measuring, inspecting, adjusting, and re-measuring a system.

## experiment log

A durable record of what was tried, why it was tried, what happened, and what was learned.

## feature

A measurable input signal used by a model.

## fine-tuning

Adapting a pretrained model to a more specific task or domain.

## foundation model

A large pretrained model that can be adapted or prompted for many downstream tasks.

## generalization

The ability of a model to perform well on new data rather than only on the training set.

## gradient

A vector of partial derivatives that describes how a function changes with respect to many parameters at once.

## gradient descent

A family of optimization methods that update parameters in the direction that most reduces loss.

## hallucination

A confident but unsupported or incorrect model output.

## human-in-the-loop

A system design pattern where people review, guide, correct, or approve model behavior.

## inference

Using a trained model to make predictions or generate outputs.

## MLOps

The engineering practice of deploying, monitoring, maintaining, and improving machine learning systems.

## model card

A concise document that describes a model's purpose, data, metrics, limitations, and intended use.

## loss function

A scalar objective used during training to measure how wrong the model's predictions currently are.

## matrix

A rectangular grid of numbers, often used to represent many examples or a linear transformation.

## overfitting

A failure mode where a model learns the training data too specifically and performs poorly on new data.

## parameter

A value learned or set within a model that affects its predictions, such as a weight or bias.

## prompt

The instruction or context provided to a language model or assistant.

## skill

A packaged reusable workflow that defines how an AI assistant should operate for a specific class of task, often including instructions, tools, constraints, context, and expected outputs.

## workflow

A repeatable sequence of steps for carrying out a task, whether or not it has been packaged into a reusable skill.

## retrieval

Finding relevant documents, items, or examples to support a task or response.

## transfer learning

Using knowledge from one training setup or domain to accelerate learning on another task.

## vector

A one-dimensional ordered collection of numbers, often representing one example, one direction, or one set of parameters.
