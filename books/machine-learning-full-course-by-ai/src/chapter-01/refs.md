# Chapter 1 Reference Notes

## Core Purpose

Establish the book's central thesis: machine learning should be learned and practiced through harness engineering, not merely passive content consumption or ad hoc prompting.

## Concepts That Must Stay Stable

- prompt versus workflow versus skill versus harness
- Codex-style skills as concrete examples of packaged workflows
- personal ML operating system
- shallow fluency as a central failure mode
- progression from learner to harness designer

## Useful Source Types for Later Expansion

- writing on human-AI collaboration and tool-mediated expertise
- software and ML engineering literature on evaluation, checklists, and technical debt
